In-Sight® Explorer 3.4.2 – Versionshinweise

18. Juli 2007  09:37:00

© Copyright 1999-2007 Cognex Corporation. Alle Rechte vorbehalten.

Übersicht

Version 3.4.2 ist eine „Service-Release“ der In-Sight Explorer-Software/Firmware. Diese Version umfasst Änderungen und Fehlerkorrekturen einschließlich Änderungen in den Dialogfeldern „Diskreter Eingang“ und „Diskreter Ausgang“, die jetzt das In-Sight 1400-E/A-Erweiterungsmodul unterstützen, und eine Aktualisierung der Firmware von Sensoren der In-Sight 5600-Serie.

Service-Release 3.4.2 wurde der Firmware/Softwareversion 3.4.1 hinzugefügt und umfasst die Korrekturen dieser Version. Diese Service-Release umfasst keine neuen Funktion. Die mit Version 3.4.0 eingeführten neuen Funktionen sind in den In-Sight ® Explorer 3.4.0-Versionshinweisen beschrieben. Die In-Sight ® Explorer 3.4.0-Versionshinweise enthalten auch Informationen über Systemanforderungen, bekannte Probleme und den Versionsverlauf.

Aktualisierte Versionshinweise, Dokumentation und mehr finden Sie im In-Sight Online Support Center unter: www.cognex.com/support/In-Sight.asp

3.4.2-Änderungen und behobene Fehler

HINWEIS: Mithilfe der (falls zutreffend) hinzugefügten Änderungsauftragsnummern ( CR# , Change Request Numbers) können die vom technischen Support von Cognex zusammengestellten bekannten Probleme besser protokolliert werden.

  • Die Dialogfelder „Einstellungen diskrete Eingänge“ und „Einstellungen diskrete Ausgänge“ wurden geändert und unterstützen jetzt das In-Sight 1400-E/A-Erweiterungsmodul.

  • Das Kuka Robot-Protokoll zur seriellen Kommunkation wurde geändert und sendet jetzt vor dem Verarbeiten von Befehlen im Native Mode die DLE-Antwort. Bisher wurde die Bestätigung erst nach der Verarbeitung des Befehls gesendet, wodurch in manchen Fällen beim Versuch, eine große Parametersatzdatei zu laden, das Zeitlimit überschritten wurde. (CR# 49787)

  • Sensoren der In-Sight 5600-Serie laden jetzt korrekt große Parametersatzdateien, die bestimmte PatMax-Funktionen enthalten. Bisher wurde beim Versuch, eine große Parametersatzdatei mit mehreren PatMax-Funktionen zu laden, die Sensorverbindung zum Netzwerk getrennt, und der nicht mehr reagierende Sensor musste durch Aus-/Einschalten neu gestartet werden. (CR# 50762)

  • Die Funktionen ScaleImage (Vision Tools --> Image) und VerifyText (Vision Tools --> OCV/OCR) wurden geändert und geben jetzt das korrekte Bild aus, wenn sie in Verbindung mit einer Teilaufnahme verwendet werden. Bisher wurde mit diesen Funktionen bei deren Verwendung mit einer Teilaufnahme in manchen Fällen ein beschädigtes Bild ausgegeben. (CR# 50931)

3.4.1-Änderungen und behobene Fehler

HINWEIS: Mithilfe der (falls zutreffend) hinzugefügten Änderungsauftragsnummern (CR# , Change Request Numbers) können die vom technischen Support von Cognex zusammengestellten bekannten Probleme besser protokolliert werden.

  • Behebt ein Problem, bei dem hochauflösende In-Sight 5603- und 5403-Sensoren unerwartet Trigger verpassen, wenn Funktionen mit hoher Prozessor- oder Arbeitsspeicherbeanspruchung in Verbindung mit der In-Sight Explorer-Anzeige von Inspektionsergebnissen ausgeführt werden. (CR# 50393)

  • In-Sight-Sensoren der Serie 5600 verursachen bei rechenintensiven Funktionen keine Bildbeschädigung mehr. Bisher wurde in manchen Fällen das Bild beschädigt, mitunter reagierte auch der Sensor nicht mehr. (CR# 50385)

  • Korrigiert ein Problem aller In-Sight 5600-Sensoren, auf denen Firmware Version 3.4.0 ausgeführt wird.   Nachdem der Benutzer das gesamte Flashdateisystem des Sensors mehrmals vollständig gefüllt und geleert hat, besteht ein sehr geringes Risiko, dass der Sensor bei einem Folgestart nicht mehr funktioniert. Die Firmware aller In-Sight 5600-Sensoren sollte auf Version 3.4.1 aktualisiert werden. Dieses Problem wirkt sich auf keine anderen In-Sight-Sensormodelle oder Firmwareversionen aus. (CR #49919)

  • Die Stringf -Funktion wurde verbessert und gibt jetzt weitere Informationszeichenfolgen im Arbeitsblatt aus: die Firmwareversion (“%V”); den Systemtyp, mit dem der Benutzer verbunden ist (“% T ”); die MAC-Adresse (“% M ”); und die Seriennummer des Sensors ("% N ").   (CR# 28899/44109)

  • Rockwell RSLogix 5000-Add-On-Profile wurden implementiert, um die Integration in Cognex In-Sight Vision-Sensoren und RSLogix 5000 zu erleichtern. Das Installationsprogramm befindet sich auf der In-Sight-Software-CD-ROM und aktualisiert eine vorhandene Installation von RSLogix 5000; RSLogix 5000 Version 15.00 oder höher muss zur Verwendung dieser Add-On-Profile installiert sein. (CR# 49914/42099)

  • Der Parameter Modus der ReadIDCode-Funktion wurde geändert und die Einstellung von Hohe Geschwindigkeit/Hoher Kontrast funktioniert jetzt korrekt für Code 128, Code 39, Code I2of5, Code 93, UPC-, EAN- und DataMatrix-Codes und ist jetzt korrekt für Postleitzahlen deaktiviert. Bei aktiviertem Kontrollkästchen Algorithmus führt die Einstellung des Parameters Modus auf Hohe Geschwindigkeit/Hoher Kontrast zu schnelleren Decodierungszeiten bei gut lesbarem Code 128, Code 39, Code I2of5, Code 93 und UPC-, EAN- und DataMatrix-Codes. Bisher war der Parameter Modus für diese Codetypen nicht aktiviert und fälschlicherweise für Postleitzahlen aktiviert. Die Decodierzeiten wurden daher nicht verbessert, wenn der Parameter auf Hohe Geschwindigkeit/Hoher Kontrast gesetzt war. (CR# 49842)

  • ValidateIDCode überprüft jetzt korrekt DoDAAC-Codes, die sowohl Alphazeichen als auch numerische Zeichen enthalten. Bisher hat ValidateIDCode DoDAAC-Codes mit Alphazeichen als ungültig bewertet. (CR# 49920)

  • Die LatchImage-Funktion gibt jetzt beim Programmieren im Emulator #ERR zurück, wenn die Einstellungen „Hoch“ oder „Breit“ des Parameters Bereich auf eine leere Zelle verweisen. Wurde bisher die LatchImage-Funktion im Emulator verwendet und haben entweder die „Hoch“- oder „Breit“-Einstellungen des Parameters Bereich auf eine leere Zelle verwiesen, hat der Emulator nicht mehr reagiert. (CR# 50019)

  • In-Sight Explorer kann jetzt umfangreichere Parametersatzdateien eines Sensors der 5600-Serie auf einem PC speichern. Bisher wurde In-Sight Explorer beim Versuch, eine über 1,8 MB große Parametersatzdatei zu speichern, vorübergehend vom Sensor getrennt, und der Parametersatz wurde nicht auf dem PC gespeichert. (CR# 49835)

  • Die GetAllTime-Funktion gibt jetzt den korrekten Wert zurück, wenn „Ergebniswarteschlange“ aktiviert ist. Bisher gab die GetAllTime-Funktion bei aktivierter „Ergebniswarteschlange“ einen falschen (niedrigeren) Wert zurück. (CR# 49722)

  • Der In-Sight-Verbindungs-Manager funktioniert jetzt korrekt, wenn er auf einem Microsoft® Windows® 2003 Server-System mit einer Gatewayadresse 0.0.0.0 ausgeführt wird. Bisher hat der In-Sight-Verbindungs-Manager nicht mehr reagiert, wenn die MAC-Adresse eines In-Sight-Sensors eingegeben und angewendet wurde. (CR# 49826)

  • In-Sight Explorer wird jetzt auf einem Windows 2003-Server, auf dem Windows Installer 3.0 nicht installiert ist, korrekt installiert. Bisher trat ein InstallShield Fehler 1603 auf, wenn In-Sight Explorer 3.4.0 auf einem Windows 2003-Server ohne Windows Installer 3.0 installiert wurde. (CR# 49827)

  • Die CalibrateImage-Funktion zeigt jetzt korrekt Ergebnisse an, wenn auf Daten einer importierten CalibrateGrid-Funktion verwiesen wird. Bisher gab die CalibrateImage-Funktion beim Verweis auf importierte CalibrateGrid-Daten #ERR zurück. (CR# 46406)

  • Die ImageMath-Funktion wurde geändert und wird jetzt korrekt angezeigt (Vision Tools --> Image). Bisher wurde die Funktion nicht korrekt aktiviert und nicht angezeigt. (CR# 43885/49630)

  • Der Bildstreifen „Ergebniswarteschlange“ bleibt bei deaktivierter Ergebniswarteschlange deaktiviert, wenn eine Jobansicht erneut geöffnet und das Layout der Jobansicht mit der Option „Layout speichern“ gespeichert wird. Bisher wurde der Bildstreifen „Ergebniswarteschlange“ bei jedem Öffnen einer Jobansicht angezeigt, unabhängig vom Status der Anzeige nach Verwendung der Option „Layout speichern“. (CR# 49266)

  • Die Parameter Startzeile und Anzahl der Zeilen der AcquireImage-Funktion geben jetzt korrekt eine Teilaufnahme zurück, wenn sie mit Farbsensoren der In-Sight 5000-Serie verwendet werden. Bisher waren diese Parameter bei Verwendung eines Farbsensors deaktiviert. (CR# 36036)

  • Gemäß EtherNet/IP-Spezifikation wurde der TTL-für-EtherNet/IP-Multicast-Datenverkehr von 7 zu 1 geändert. Diese Änderung beschränkt den Multicast-Datenverkehr auf ein einzelnes Subnetz. (CR# 50000)
