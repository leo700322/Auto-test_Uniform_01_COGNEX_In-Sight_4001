In-Sight® 浏览器 3.4.2 发行说明

2007 年 7 月 18 日 上午 9:37

1999-2007年Cognex Corporation 版权所有。

概述

版本 3.4.2 是 In-Sight 浏览器固件/软件的服务版本。此版本提供对各种已知问题的更改和修复，其中包括对“离散输入设置”和“离散输出设置”对话框进行修改以添加对 In-Sight 1400 I/O 扩展模块的支持以及对 In-Sight 5600 系列传感器的固件进行更新。

服务版本 3.4.2 添加到了固件/软件版本 3.4.1 中，包含该版本中提供的修复。此服务版本不包含任何新功能；版本 3.4.0 引入了新功能，In-Sight ® 浏览器 3.4.0 发行说明中对这些新功能做了介绍。要查看有关“系统要求”、“已知问题”和“发行历史”的信息，请参阅 In-Sight ® 浏览器 3.4.0 发行说明。

有关更新的发行说明、文档及其它信息，请访问“In-Sight 联机支持中心”，其网址为：www.cognex.com/support/In-Sight.asp

3.4.2 更改与修复

注意：为改进对“ Cognex 技术支持”报告的“已知问题”的跟踪，（在适用的地方）新增了“更改请求”号码 ( CR# )。

  • 对“离散输入设置”和“离散输出设置”对话框进行了修改，以提供对 In-Sight 1400 I/O 扩展模块的支持。

  • 对 Kuka 机器人串行通信协议进行了修改，以先发送 DLE 响应，再处理“本机模式”命令。以前，处理“本机模式”命令后才会发送确认，这会导致机器人在尝试加载大型作业文件时超时。(CR# 49787)

  • 现在，In-Sight 5600 系列传感器可以正确地加载包含某些 PatMax 函数的大型作业文件。以前，在某些情况下，尝试加载包含多个 PatMax 函数的大型作业文件时，传感器会断开与网络的连接并且失去响应，提示对传感器重新加电。(CR# 50762)

  • ScaleImage（视觉工具 --> 图像）和 VerifyText（视觉工具 --> OCV/OCR）函数进行了修改，以在与部分采集联用时输出正确的图像。以前，对部分采集的图像使用这些函数时会输出损坏的结果图像。(CR# 50931)

3.4.1 更改与修复

注意：为改进对“ Cognex 技术支持”报告的“已知问题”的跟踪，（在适用的地方）新增了“更改请求”号码 ( CR# )。

  • 更正了 In-Sight 5603 和 5403 高分辨率传感器存在的一个问题：执行计算或内存占用高的功能并连同使用 In-Sight 浏览器视图检查结果时，它们会意外地错过采集触发。(CR# 50393)

  • 在 In-Sight 5600 系列传感器上，执行需要进行大量计算的功能时不再发生图像损坏的情况。以前，图像可能会损坏，而且在某些情况下传感器可能会变得不做出响应。(CR# 50385)

  • 更正了所有运行固件版本 3.4.0 的 In-Sight 5600 传感器都存在的一个问题。   如果用户曾多次完全填充和擦除传感器上的整个闪存文件系统，则之后重新启动时会导致传感器无法运行的风险是很小的。所有 In-Sight 5600 传感器的固件都应升级到版本 3.4.1。此问题不影响任何其它 In-Sight 传感器型号或固件修订版。(CR# 49919)

  • 增强了 Stringf 函数，使其能够将附加信息字符串返回到电子表格：固件版本 ("%V")；用户连接到的系统类型 ("% T ")；MAC 地址 ("% M ") 和传感器的序列号 ("% N ")。   ( CR# 28899/44109)

  • 实现了 Rockwell RSLogix 5000 附加配置文件，以帮助简化 Cognex In-Sight 视觉传感器与 RSLogix 5000 之间的集成。安装程序位于 In-Sight 软件光盘中，它将更新现有 RSLogix 5000 安装；要使用这些附加配置文件，必须安装 RSLogix 5000 版本 15.00 或更高版本。( CR# 49914/42099)

  • 修改了 ReadIDCode 函数的 Mode 参数，现在可以正常地为 Code 128、Code 39、Code I2of5、Code 93、UPC、EAN 和 DataMatrix 代码启用高速度/高对比度设置，同时也可正常地为“邮政编码”禁用该设置。启用算法复选框时，将模式参数设置为高速度/高对比度可缩短印制清晰的 Code 128、Code 39、Code I2of5、Code 93、UPC、EAN 及 DataMatrix 代码的解码时间。在先前版本中，没有为这些代码类型启用模式参数，并为“邮政编码”不正确地启用了该参数，导致该参数设置为高速度/高对比度时并未使解码时间得到缩短。( CR# 49842)

  • ValidateIDCode 现在会正确地验证同时包含字母和数字字符的 DoDAAC 代码。在先前版本中， ValidateIDCode 会将包含字母字符的 DoDAAC 代码报告为无效代码。( CR# 49920)

  • 现在，在仿真器中使用 LatchImage 函数进行编程时，如果区域参数的“高”或“宽”设置引用空单元格，将会返回错误代码。在先前版本中，在仿真器中使用 LatchImage 函数时，如果区域参数的“高”或“宽”设置之一引用了空单元格，仿真器将不做响应。( CR# 50019)

  • 现在，In-Sight 浏览器可将更大的作业文件从 5600 系列传感器保存到 PC。在先前版本中，尝试保存超过 1.8MB 的作业文件时，In-Sight 浏览器会暂时断开与传感器的连接，作业将不会保存到 PC。(CR# 49835)

  • 现在启用“结果队列”时 GetAllTime 函数会返回正确的值。在先前版本中，启用“结果队列”时 GetAllTime 函数会返回不正确（较低）的值。(CR# 49722)

  • 现在，在网关地址设置为 0.0.0.0 的 Microsoft® Windows® 2003 Server 系统上运行时，“In-Sight 连接管理器”可以正常地工作。在先前版本中，输入并应用 In-Sight 传感器的 MAC 地址后，“In-Sight 连接管理器”将不做响应。(CR# 49826)

  • In-Sight 浏览器现在可以正确地安装在未安装 Windows Installer 3.0 的 Windows 2003 Server 上。   在先前版本中，在未安装 Windows Installer 3.0 的 Windows 2003 Server 上安装 In-Sight 浏览器 3.4.0 时会出现 InstallShield 1603 错误。(CR# 49827)

  • 现在 CalibrateImage 函数从导入的 CalibrateGrid 函数引用数据时会正确地显示结果。在先前版本中，CalibrateImage 引用导入的 CalibrateGrid 数据时会返回错误代码。(CR# 46406)

  • 修改了 ImageMath 函数，现在已可正确地显示（视觉工具 --> 图像）。在先前版本中，该函数未被正常启用，不会显示出来。(CR# 43885/49630)

  • 如果关闭“结果队列”显示并使用“保存布局”选项保存“作业视图”的布局，则重新打开“作业视图”时“结果队列”胶片仍为关闭状态。在先前版本中，只要打开“作业视图”，无论使用“保存布局”选项之后的显示状态为何，均会显示“结果队列”胶片。(CR# 49266)

  • 现在，在 In-Sight 5000 系列彩色传感器上使用 AcquireImage 函数的开始行和行数参数时，它们会正常地返回部分采集。在先前版本中，使用彩色传感器时会禁用这些参数。(CR# 36036)

  • 已遵循 EtherNet/IP 规范将 EtherNet/IP 组播通信量的 TTL 从 7 更改为 1。此更改将组播通信量限制为单个子网。(CR# 50000)
