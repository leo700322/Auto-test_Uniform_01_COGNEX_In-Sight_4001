In-Sight® 익스플로러 3.4.2 출시 안내

2007년 7월 18일 9:37 AM

© Copyright 1999-2007 Cognex Corporation. All rights reserved.

개요

릴리스 3.4.2는 In-Sight 익스플로러 펌웨어/소프트웨어의 서비스 릴리스입니다. 이 릴리스는 In-Sight 1400 I/O 확장 모듈에 대한 지원을 추가하기 위해 접점 입출력 설정 대화 상자의 수정을 포함하여 다양한 알려진 문제에 대한 변경 및 수정과 In-Sight 5600 시리즈 센서의 펌웨어 업데이트를 제공합니다.

서비스 릴리스 3.4.2는 펌웨어/소프트웨어 릴리스 3.4.1에 추가되었고, 해당 릴리스에 제공되었던 수정 사항도 포함되어 있습니다. 이 서비스 릴리스에는 새로운 기능에 대한 사항이 포함되어 있지 않으며, 릴리스 3.4.0에서 소개된 새 기능은 In-Sight® 익스플로러 3.4.0 출시 안내서에 설명되어 있습니다. 시스템 요건, 알려진 문제점 및 출시 내역에 대한 정보를 보려면 In-Sight® 익스플로러 3.4.0 출시 안내서를 참조하십시오.

업데이트된 출시 안내서, 문서 등에 대한 정보는 In-Sight 온라인 지원 센터 www.cognex.com/support/In-Sight.asp로 로그인하십시오.

3.4.2 변경 및 수정 사항

참고: 변경 요청 번호( CR# )가 포함되어(해당되는 경우) Cognex 기술 지원을 통해 보고된 알려진 문제점의 추적을 향상시킵니다.

  • 접점 입력 설정 및 접점 출력 설정 대화 상자가 In-Sight 1400 I/O 확장 모듈에 대한 지원을 제공하도록 수정되었습니다.

  • 고유 모드 명령을 처리하기 전에 DLE 반응을 발송하도록 Kuka 로봇 직렬 통신 프로토콜이 수정되었습니다. 종전에는 명령이 처리된 후까지 인정이 발송되지 않았기 때문에 큰 작업 파일을 로드하려고 시도하는 동안 로봇이 타임아웃될 수 있었습니다. (CR #49787)

  • In-Sight 5600 시리즈 센서는 이제 특정 PatMax 함수가 포함된 큰 작업 파일을 정확하게 로드합니다. 종전에는 복수의 PatMax 함수가 포함된 큰 작업 파일을 로드하려고 시도할 때, 센서가 네트워크로부터 분리되고 반응을 하지 않게 되어 센서의 전력 순환을 야기하는 경우도 있었습니다. (CR #50762)

  • ScaleImage(비전 도구 --> 이미지) 및 VerifyText(비전 도구 --> OCV/OCR) 기능이 부분 인식과 함께 사용할 경우 올바른 이미지를 출력하기 위해 수정되었습니다. 이전에는 이러한 기능이 부분 인식 이미지와 함께 사용될 경우 손상된 결과 이미지를 출력하는 경우가 있었습니다. (CR# 50931)

3.4.1 변경 및 수정 사항

참고: 변경 요청 번호( CR# )가 포함되어(해당되는 경우) Cognex 기술 지원을 통해 보고된 알려진 문제점의 추적을 향상시킵니다.

  • In-Sight 익스플로러 검사 결과 보기를 사용하는 동시에 계산 또는 메모리 중심 기능을 실행하는 경우, In-Sight 5603 및 5403 고해상도 센서가 예기치 않게 인식 트리거를 누락할 수 있는 문제를 수정합니다. (CR #50393)

  • In-Sight 5600 시리즈 센서에서는 계산 중심적 기능 실행 시, 더 이상의 이미지 충돌이 발생하지 않습니다. 예전에는 이미지가 손상되거나 센서가 응답하지 않는 경우가 있었습니다. (CR# 50385)

  • 펌웨어 버전 3.4.0을 실행하는 모든 In-Sight 5600 센서의 문제점을 해결합니다.   센서에 있는 전체 플래시 파일 시스템이 완전히 채워져 여러 번 삭제한 경우, 흔한 경우는 아니지만 다음 번 재부팅할 때 센서가 작동하지 않을 수 있습니다. 모든 In-Sight 5600 센서는 펌웨어를 버전 3.4.1로 업그레이드해야 합니다. 이 문제점은 다른 In-Sight 센서 모델이나 펌웨어 개정에 영향을 미치지 않습니다. (CR# 49919)

  • Stringf 함수는 펌웨어 버전(“%V”), 사용자가 연결된 시스템 유형(“% T ”), MAC 주소(“% M ”) 및 센서의 일련 번호("% N ") 등 추가 정보 문자열을 스프레드시트에 반환하도록 향상되었습니다.   ( CR# 28899/44109)

  • Rockwell RSLogix 5000 Add-On Profiles가 Cognex In-Sight 비전 센서와 RSLogix 5000 사이의 통합을 간소화하도록 구현되었습니다. 설치 프로그램은 In-Sight 소프트웨어 CD-ROM에 들어 있으며 RSLogix 5000의 기존 설치를 업데이트합니다. 이 Add-On Profiles을 사용하려면 RSLogix 5000 버전 15.00 이상을 설치해야 합니다. ( CR# 49914/42099)

  • ReadIDCode 함수 Mode 매개변수가 수정되었으며 고속/고대비 설정이 이제 코드 128, 코드 39, 코드 I2of5, 코드 93, UPC, EAN 및 DataMatrix 코드에 대해 올바르게 활성화되며 우편 번호에 대해서도 올바르게 비활성화됩니다. 알고리즘 체크박스가 활성화되면 모드 매개변수를 고속/고대비로 설정하면 잘 인쇄된 코드 128, 코드 39, 코드 I2of5, 코드 93, UPC, EAN 및 DataMatrix 코드에 대한 디코딩 시간이 더 빨라질 수 있습니다. 이전에는 모드 매개변수가 이러한 코드 유형에 대해 활성화되지 않았으며 우편 코드에 대해 잘못 활성화되어 매개변수가 고속/고대비로 설정했을 때 디코딩 시간이 향상되지 않았습니다. ( CR# 49842)

  • ValidateIDCode 가 이제 알파벳 문자와 숫자 문자를 모두 포함하고 있는 DoDAAC 코드를 올바르게 확인합니다. 이전에는 ValidateIDCode 가 알파벳 문자와 숫자 문자를 모두 포함하고 있는 DoDAAC 코드를 잘못된 것으로 보고합니다. ( CR# 49920)

  • LatchImage 함수는 에뮬레이터에서 프로그래밍하면 이제 Region 매개변수의 높음 또는 넓음 설정이 빈 셀을 참조하는 경우 #ERR을 반환합니다. 이전에는 LatchImage 함수가 에뮬레이터에서 사용될 때 Region 매개변수의 높음 또는 넓음 설정이 빈 셀을 참조한 경우 에뮬레이터가 응답하지 않았습니다. ( CR# 50019)

  • 이제 In-Sight 익스플로러에서 5600 시리즈에서 PC로 더 큰 작업 파일을 저장할 수 있습니다. 이전에는 1.8MB 이상의 작업 파일을 저장하려고 할 때 In-Sight 익스플로러가 일시적으로 센서에서 연결이 끊어지고 작업은 PC에 저장되지 않았습니다. (CR# 49835)

  • GetAllTime 함수는 이제 결과 대기열이 활성화될 때 올바른 값을 반환합니다. 이전에는 GetAllTime 함수가 결과 대기열이 활성화되었을 때 올바르지 않은(낮은) 값을 반환했습니다. (CR# 49722)

  • In-Sight 연결 관리자가 게이트웨이 주소가 0.0.0.0으로 설정된 Microsoft® Windows® 2003 Server 시스템에서 실행할 때 올바르게 작동합니다. 이전에는 In-Sight 센서의 MAC 주소를 입력하고 적용한 후에 In-Sight 연결 관리자가 응답하지 않았습니다. (CR# 49826)

  • In-Sight 익스플로러는 이제 Windows Installer 3.0이 설치되지 않은 Windows 2003 Server에 올바르게 설치됩니다.   이전에는 Windows Installer 3.0 없이 Windows 2003 Server에 In-Sight 익스플로러 3.4.0을 설치할 때 InstallShield 1603 오류가 나타났습니다. (CR# 49827)

  • CalibrateImage 함수는 이제 가져온 CalibrateGrid 함수의 데이터를 참조할 때 결과를 올바르게 표시합니다. 이전에는 가져온 CalibrateGrid 데이터를 참조할 때 CalibrateImage가 #ERR을 반환했습니다. (CR# 46406)

  • ImageMath 함수가 수정되었으며 이제 올바르게 표시됩니다(비전 도구 --> 이미지). 이전에는 함수가 올바르게 활성화되지 않고 표시되지 않았습니다. (CR# 43885/49630)

  • 결과 대기열 디스플레이가 꺼져 있고 작업 보기 레이아웃을 레이아웃 저장 옵션을 사용하여 저장한 경우, 작업 보기를 다시 열면 결과 대기열 영사 슬라이드가 켜지지 않습니다. 이전에는 레이아웃 저장 옵션을 사용한 후의 디스플레이 상태에 관계 없이 작업 보기가 열렸을 때마다 결과 대기열 영사 슬라이드가 표시되었습니다. (CR# 49266)

  • AcquireImage 함수의 시작 행과 행 수 매개변수는 이제 In-Sight 5000 시리즈 컬러 센서에서 사용할 때 부분 인식을 제대로 반환합니다. 이전에는 컬러 센서를 사용할 때 이러한 매개변수가 비활성화되었습니다. (CR# 36036)

  • EtherNet/IP 사양에 따라 EtherNet/IP 멀티캐스트 트래픽용 TTL이 7에서 1로 변경되었습니다. 이 변경은 멀티캐스트 트래픽을 단일 서브넷으로 제한합니다. (CR# 50000)
