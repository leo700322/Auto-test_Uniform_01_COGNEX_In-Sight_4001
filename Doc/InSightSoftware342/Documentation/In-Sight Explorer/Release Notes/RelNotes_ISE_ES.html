Resumen

La versión 3.4.2 es una versión de servicio del software/firmware de In-Sight Explorer. Esta versión contiene cambios y correcciones de varios problemas conocidos, incluyendo modificaciones en los cuadros de diálogo Opciones de entrada discreta y Opciones de salida discreta para añadir compatibilidad con el módulo de expansión de E/S In-Sight 1400, así como una actualización del firmware de los sensores In-Sight de la serie 5600.

La versión de servicio 3.4.2 es una ampliación de la versión de firmware y software 3.4.1, por lo que contiene las mismas correcciones que esta última. Esta versión de servicio no incluye ninguna función nueva; en la versión 3.4.0 se incorporó una nueva función, descrita en las Notas de la versión 3.4.0 de In-Sight ® Explorer. Para obtener información sobre los requisitos del sistema, problemas conocidos y el historial de versiones, consulte las Notas de la versión 3.4.0 de In-Sight® Explorer.

Para obtener las notas, documentación e información más recientes sobre esta versión, visite el centro de asistencia en línea de In-Sight en Internet: www.cognex.com/support/In-Sight.asp

Modificaciones y correcciones en la versión 3.4.2

NOTA: Se han incluido (donde corresponda) números de solicitud de modificación ( CR# ) para mejorar el seguimiento de los problemas conocidos registrados a través de la Asistencia técnica de Cognex .

  • Los cuadros de diálogo Opciones de entrada discreta y Opciones de salida discreta han sido modificados para proporcionar compatibilidad con el módulo de expansión de E/S In-Sight 1400.

  • Se ha modificado el protocolo de comunicación serie para poder enviar la respuesta DLE antes de procesar comandos del modo nativo. En versiones anteriores, la confirmación no se enviaba hasta después de haberse procesado el comando, lo que podía provocar que el robot superara el tiempo de espera al intentar cargar un archivo de tareas de gran tamaño. (CR# 49787)

  • Ahora los sensores In-Sight de la serie 5600 cargan correctamente archivos de tareas de gran tamaño que contienen determinadas funciones PatMax. Hasta ahora, en determinadas circunstancias podía ocurrir que, al intentar cargar un archivo de tareas de gran tamaño con varias funciones PatMax, el sensor se desconectase de la red, dejara de responder y solicitase ser apagado y encendido. (CR# 50762)

  • Las funciones ScaleImage (Herramientas de visión --> Imagen) y VerifyText (Herramientas de visión --> OCV/OCR) se han modificado para que generen una imagen correcta si se utilizan en combinación con una adquisición parcial. Anteriormente, si estas funciones se utilizaban con una imagen parcialmente adquirida se podía dar lugar a una imagen resultante corrupta. (CR# 50931)

Modificaciones y correcciones en la versión 3.4.1

NOTA: Se han incluido (donde corresponda) números de solicitud de modificación ( CR# ) para mejorar el seguimiento de los problemas conocidos registrados a través de la Asistencia técnica de Cognex .

  • Corrige un problema consistente en que los sensores de alta resolución In-Sight 5603 y 5403 podían perder inesperadamente disparadores de adquisición si se ejecutaban funciones de cálculo intensivo o de uso intensivo de memoria mientras se utilizaba In-Sight Explorer para visualizar resultados de inspección. (CR# 50393)

  • Al ejecutar funciones de cálculo intensivo con sensores de la serie In-Sight 5600, las imágenes ya no resultan dañadas. Anteriormente, las imágenes podían resultar dañadas y, en algunos casos, el sensor dejaba de responder. (CR# 50385)

  • Corrige un problema para todos los sensores In-Sight 5600 con la versión de firmware 3.4.0.   Si el usuario ha completado y borrado completamente, varias veces, todo el sistema de archivos flash en el sensor, el riesgo de que el sensor quede inutilizable en un reinicio posterior es muy reducido. Todos los sensores In-Sight 5600 deben actualizarse a la versión de firmware 3.4.1. Este problema no afecta a ningún otro modelo de sensor o versión de firmware de In-Sight. (CR# 49919)

  • La función Stringf se ha mejorado para que devuelva cadenas de información adicional a la hoja de cálculo: la versión de firmware (“%V”); el tipo de sistema al que el usuario está conectado (“% T ”); la dirección MAC (“% M ”) y el número de serie del sensor ("% N ").   ( CR# 28899/44109)

  • Se han implementado perfiles de complementos (Add-On Profiles o AOP) para Rockwell RSLogix 5000 con el fin de simplificar la integración entre los sensores de visión In-Sight de Cognex y RSLogix 5000. El programa instalador se encuentra en el CD-ROM del software de In-Sight y actualizará la instalación existente de RSLogix 5000; para utilizar estos perfiles de complementos, debe tener instalada la versión 15.00 o superior de RSLogix 5000. ( CR# 49914/42099)

  • Se ha modificado el parámetro Modo de la función ReadIDCode y la opción Alta velocidad/Alto contraste ahora está correctamente habilitada para los códigos Code 128, Code 39, Code I2of5, Code 93, UPC, EAN y DataMatrix y está correctamente deshabilitada para códigos postales. Si se marca la casilla de verificación Algoritmo, al ajustar el parámetro Modo a Alta velocidad/Alto contraste se pueden lograr tiempos de decodificación más rápidos para los códigos Code 128, Code 39, Code I2of5, Code 93, UPC, EAN y DataMatrix correctamente impresos. Anteriormente el parámetro Modo no estaba habilitado para este tipo de códigos, pero sí estaba incorrectamente habilitado para los códigos postales. Esto no producía ninguna mejora en los tiempos de decodificación cuando el parámetro se ajustaba a Alta velocidad/Alto contraste. ( CR# 49842)

  • ValidateIDCode ahora valida correctamente los códigos DoDAAC que contienen caracteres alfanuméricos. Anteriormente, ValidateIDCode informaba de los códigos DoDAAC que contenían caracteres alfabéticos como códigos no válidos. ( CR# 49920)

  • La función LatchImage, al programarse en el emulador, ahora devuelve #ERR si las opciones Alto o Ancho del parámetro Región hacen referencia a una celda vacía. Anteriormente, si la función LatchImage se utilizaba en el emulador y las opciones Alto o Ancho del parámetro Región hacían referencia a una celda vacía, el emulador dejaba de responder. ( CR# 50019)

  • In-Sight Explorer ahora puede guardar archivos de tareas de mayor tamaño de un sensor de la serie 5600 en un PC. Anteriormente, al intentar guardar un archivo de tarea superior a 1,8 MB, In-Sight Explorer se desconectaba temporalmente del sensor y la tarea no se podía guardar en el PC. (CR# 49835)

  • La función GetAllTime ahora devuelve el valor correcto cuando se habilita Cola de resultados. Anteriormente, la función GetAllTime devolvía un valor incorrecto (inferior) cuando se habilitaba Cola de resultados. (CR# 49722)

  • El Administrador de conexión de In-Sight ahora funciona correctamente al ejecutarse en un servidor del sistema Microsoft® Windows® 2003 con una dirección de puerta de enlace 0.0.0.0. Anteriormente, el Administrador de conexión de In-Sight dejaba de responder después de que se introdujera y aplicara una dirección MAC del sensor In-Sight. (CR# 49826)

  • In-Sight Explorer ahora se instala correctamente en un servidor Windows 2003 que no tenga instalado Windows Installer 3.0.   Anteriormente, se mostraba un error de InstallShield 1603 al instalar In-Sight Explorer 3.4.0 en un servidor Windows 2003 Server sin Windows Installer 3.0. (CR# 49827)

  • La función CalibrateImage ahora muestra correctamente los resultados al hacer referencia a datos de una función CalibrateGrid importada. Anteriormente, CalibrateImage devolvía #ERR al hacer referencia a datos de la función CalibrateGrid importada. (CR# 46406)

  • La función ImageMath se ha modificado y ahora se muestra correctamente (Herramientas de visión --> Imagen). Anteriormente, la función no estaba correctamente habilitada y no se mostraba. (CR# 43885/49630)

  • La película Cola de resultados permanecerá desactivada si se vuelve a abrir una vista de tareas, siempre que la pantalla Cola de resultados se haya desactivado y la vista de tareas se haya guardado usando la opción Guardar diseño. Anteriormente, la película Cola de resultados se mostraba siempre que una vista de tareas se abría, independientemente del estado de la pantalla tras el uso de la opción Guardar diseño. (CR# 49266)

  • Los parámetros Fila inicial y Cantidad de filas de la función AcquireImage ahora devuelven correctamente una adquisición parcial al utilizarse en sensores de color de la serie In-Sight 5000. Anteriormente, estos parámetros se deshabilitaban al utilizar un sensor de color. (CR# 36036)

  • Siguiendo la especificación EtherNet/IP, TTL para el tráfico de multidifusión EtherNet/IP se ha cambiado de 7 a 1. Este cambio limita el tráfico de multidifusión a una única subred. (CR# 50000)
