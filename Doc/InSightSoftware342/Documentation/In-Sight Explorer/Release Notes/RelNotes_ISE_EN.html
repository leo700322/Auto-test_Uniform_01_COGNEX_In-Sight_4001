Overview

Release 3.4.2 is a service release of In-Sight Explorer firmware/software. This release provides changes and fixes to various known issues, including modifications to the Discrete Input and Output Settings dialogs to add support for the In-Sight 1400 I/O Expansion Module and an update to the firmware of In-Sight 5600 Series Sensors.

Service release 3.4.2 was added to firmware/software release 3.4.1, and contains fixes provided in that release. This service release does not include any new functionality; release 3.4.0 introduced new functionality, which is described in the In-Sight® Explorer 3.4.0 Release Notes. To view information about System Requirements, Known Issues and Release History, please see the In-Sight® Explorer 3.4.0 Release Notes.

For updated release notes, documentation and more, go to the In-Sight Online Support Center at: www.cognex.com/support/In-Sight.asp

3.4.2 Changes & Fixes

NOTE: Change Request numbers ( CR# ) have been included (where applicable) to improve tracking of Known Issues reported via Cognex Technical Support.

  • The Discrete Input Settings and Discrete Output Settings dialogs have been modified to provide support for the In-Sight 1400 I/O Expansion Module.

  • Kuka robot serial communication protocol has been modified to send the DLE response before processing Native Mode commands. Previously, the acknowledgment was not sent until after the command had been processed, which could cause the robot to timeout while attempting to load a large job file. (CR# 49787)

  • In-Sight 5600 series sensors now correctly load large job files that contain certain PatMax functions. Previously, in certain circumstances, when attempting to load a large job file that contained multiple PatMax functions, the sensor would disconnect from the network and become unresponsive, prompting a power cycle of the sensor. (CR# 50762)

  • The ScaleImage (Vision Tools --> Image) and VerifyText (Vision Tools --> OCV/OCR) functions have been modified to output the correct image when used in conjunction with a partial acquisition. Previously, these functions, when used with a partially acquired image, could output a corrupted result image. (CR#  50931)

3.4.1 Changes & Fixes

NOTE: Change Request numbers ( CR# ) have been included (where applicable) to improve tracking of Known Issues reported via Cognex Technical Support.

  • Corrects an issue where In-Sight 5603 and 5403 high-resolution sensors could unexpectedly miss acquisition triggers when executing computation or memory-intensive functions in conjunction with using In-Sight Explorer view inspection results. (CR# 50393)

  • On In-Sight 5600 series sensors, when executing computation-intensive functions, image corruption no longer occurs. Previously, the image could become corrupted, and in some cases the sensor could become unresponsive. (CR# 50385)

  • Corrects an issue for all In-Sight 5600 sensors running firmware version 3.4.0.  If the user had completely filled and erased the entire flash file system on the sensor multiple times, there is a very small risk that the sensor could be rendered inoperable on a subsequent reboot. All In-Sight 5600 sensors should be firmware upgraded to version 3.4.1. This issue does not affect any other In-Sight sensor models or firmware revisions. (CR# 49919)

  • The Stringf function has been enhanced to return additional information strings to the spreadsheet: the firmware version (“%V”); the system type to which the user is connected (“% T ”); the MAC address (“% M ”); and the sensor's serial number ("% N ").  ( CR# 28899/44109)

  • Rockwell RSLogix 5000 Add-On Profiles have been implemented to help simplify the integration between Cognex In-Sight vision sensors and RSLogix 5000. The installer is located on the In-Sight Software CD-ROM and will update an existing installation of RSLogix 5000; RSLogix 5000 version 15.00 or greater must be installed in order to use these Add-On Profiles. ( CR# 49914/42099)

  • The ReadIDCode function Mode parameter has been modified and the High Speed/High Contrast setting is now correctly enabled for Code 128, Code 39, Code I2of5, Code 93, UPC, EAN and Data Matrix codes, while also being correctly disabled for Postal Codes. When the Algorithm checkbox is enabled, setting the Mode parameter to High Speed/High Contrast may yield faster decode times for well-printed Code 128, Code 39, Code I2of5, Code 93, UPC, EAN and Data Matrix codes. Previously, the Mode parameter was not enabled for those code types, and incorrectly enabled for Postal Codes, which did not result in any decode time improvement when the parameter was set to High Speed/High Contrast. ( CR# 49842)

  •  ValidateIDCode now correctly validates DoDAAC codes containing both alpha and numeric characters. Previously, ValidateIDCode would report DoDAAC codes containing alpha characters as invalid. ( CR# 49920)

  • The LatchImage function, when being programmed in the emulator, will now return #ERR if the High or Wide settings of the Region parameter reference an empty cell. Previously, if the LatchImage function was used in the emulator and either the High or Wide settings of the Region parameter referenced an empty cell, the emulator would become unresponsive. ( CR# 50019)

  • In-Sight Explorer now can save larger job files from a 5600 series sensor to a PC. Previously, when trying to save a job file over 1.8MB, In-Sight Explorer would temporarily disconnect from the sensor and the job would not be saved to the PC. (CR# 49835)

  • The GetAllTime function now returns the correct value when Results Queue is enabled. Previously, GetAllTime function returned an incorrect (lower) value when Results Queue was enabled. (CR# 49722)

  • In-Sight Connection Manager now functions correctly when running on a Microsoft® Windows® 2003 Server system with a gateway address set to 0.0.0.0. Previously, In-Sight Connection Manager became unresponsive after an In-Sight sensor’s MAC address was entered and applied. (CR# 49826)

  • In-Sight Explorer now installs correctly on a Windows Server 2003 that does not have Windows Installer 3.0 installed.  Previously, an InstallShield 1603 error would appear when installing In-Sight Explorer 3.4.0 on a Windows Server 2003 without Windows Installer 3.0. (CR# 49827)

  • The CalibrateImage function now correctly displays results when referencing data from an imported CalibrateGrid function. Previously, CalibrateImage returned #ERR when referencing imported CalibrateGrid data. (CR# 46406)

  • The ImageMath function has been modified and now is displayed correctly (Vision Tools --> Image). Previously, the function was not properly enabled and was not displayed. (CR# 43885/49630)

  • The Results Queue filmstrip will remain off when a Job View is re-opened if the Results Queue display is turned off and the Job View's layout is saved using the Save Layout option. Previously, the Results Queue filmstrip would be displayed whenever a Job View was opened, regardless of the state of the display after using the Save Layout option. (CR# 49266)

  • The Start Row and Number of Rows parameters of the AcquireImage function now properly return a partial acquisition when used on In-Sight 5000 series color sensors. Previously, these parameters were disabled when using a color sensor. (CR# 36036)

  • Following the EtherNet/IP Specification, the TTL for EtherNet/IP multicast traffic has been changed from 7 to 1. This change limits multicast traffic to a single subnet. (CR# 50000)
