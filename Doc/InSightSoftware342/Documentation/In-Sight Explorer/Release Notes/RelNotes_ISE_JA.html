2007年7月31日 7時32分

概要

リリース 3.4.2 は、In-Sight Explorer ソフトウェアおよびファームウェアのサービスリリースです。このリリースでは、既知の問題を修正しているほか、新たに I/O 拡張モジュール CIO-1400 のサポートを追加しています。

リリース 3.4.0 で導入された新機能は、In-Sight Explorer 3.4.0 リリースノートに記載されています。また、システム要件、既知の問題およびリリース履歴についても、In-Sight Explorer 3.4.0 のリリースノートをご参照ください。

更新されたリリースノートやマニュアル等は、In-Sight オンラインサポートセンター で入手できます。

リリース 3.4.2 での修正点

注: Cognexテクニカルサポートを介して報告された既知の問題を追跡しやすくするために、Change Request番号（CR#）が付与されています。

  • I/O 拡張モジュール CIO-1400 のサポートに伴い、[ディスクリート入力] および [ディスクリート出力] 設定ダイアログが変更されました。

  • Kuka ロボットのシリアル通信プロトコルが変更され、ネイティブモードコマンドを処理する前に DLE レスポンスを送信できるようになりました。以前は、コマンド処理後に応答していたため、大きなジョブファイルをロードしようと試みた場合、ロボットがタイムアウトを起こすことがありました。（CR# 49787）

  • 特定の PatMax 関数を含む容量の大きなジョブファイルを In-Sight 5600 センサシリーズに正しくロードできるようになりました。以前は、特定の状況下で複数の PatMax 関数を含む大きなジョブファイルのロードを試みた場合に、センサがネットワークから切断され、無応答になり、再起動する必要がありました。（CR# 50762）

  • 部分取り込みした画像に対して ScaleImage 関数（[ビジョンツール] → [画像フィルタ]）や VerifyText 関数（[ビジョンツール] → [文字照合/認識]）を実行したときに崩れた画像が出力される不具合を修正しました。（CR# 50931）

リリース 3.4.1 での修正点

注: Cognexテクニカルサポートを介して報告された既知の問題を追跡しやすくするために、Change Request番号（CR#）が付与されています。

  • 高解像度センサ In-Sight 5603 および In-Sight 5403 が特定の状況下で画像取り込みに失敗する問題を修正しました。以前は、高負荷状態になったこれらのセンサのジョブビューを In-Sight Explorer で表示しているとき、予期しない画像取り込みミスが発生していました。（CR# 50393）

  • In-Sight 5600 シリーズセンサの画像が破壊される問題を修正しました。以前は、演算負荷の高い関数を実行すると市松模様のような画像になり、場合によっては、センサが無応答になることがありました。（CR# 50385）

  • ファームウェアバージョン 3.4.0 を搭載した In-Sight 5600 シリーズの問題を修正しました。センサ上のフラッシュファイルシステムを完全に使い切ってから全消去するサイクルを複数回繰り返すと、センサの再起動時に動作不能になることがあります。In-Sight 5600 シリーズのファームウェアは、バージョン 3.4.2 以上にアップグレードしてください。なお、この問題は In-Sight 5600 シリーズ以外のセンサでは発生しません。（CR# 49919）

  • Stringf 関数が強化され、次の情報文字列をスプレッドシートに返すことができるようになりました。ファームウェアバージョン（%V）、システムタイプ（%T）、MAC アドレス（%M）、センサのシリアル番号（%N）を取得可能です。（CR# 28899, 44109）

  • In-Sight ビジョンセンサと Rockwell RSLogix 5000 を簡単に統合する RSLogix 5000 アドオンプロファイルが用意されました。このインストーラは In-Sight ソフトウェア CD-ROM に格納されており、既存の RSLogix 5000 インストールを更新します。このアドオンプロファイルを使用するには、RSLogix 5000 のバージョン 15.00 以上が必要です。（CR# 49914, 42099）

  • ReadIDCode 関数の [モード] パラメータが変更されました。Code 128、Code 39、Code I2of5、Code 93、UPC、EAN またはデータマトリックスを選択しているときに [モード] パラメータが有効となり、ポスタルコードを選択しているときは無効になります。[アルゴリズム] チェックボックスを ON にし、[モード] パラメータを [高速/高コントラスト] に設定すると、はっきりと印字された Code 128、Code 39、Code I2of5、Code 93、UPC、EAN およびデータマトリックスのデコード時間を短縮することができます。以前は、これらのコードダイプを選択しても [モード] パラメータが有効にならず、ポスタルコードを選択したときに誤って有効にしていました。そのため、[高速/高コントラスト] を指定しても、デコード時間は改善されませんでした。（CR# 49842）

  • ValidateIDCode 関数は、アルファベットと数字の両方を含む DoDAAC コードを正しく検証できるようになりました。以前は、ValidateIDCode 関数は、アルファベットを含む DoDAAC コードを無効として報告していました。（CR# 49920）

  • LatchImage 関数の [高さ] または [幅] パラメータで空白セルを参照しているとき、正しく #ERR が返されるようになりました。以前は、領域の高さまたは幅の参照先が空白セルの場合、In-Sight エミュレータがハングアップしていました。（CR# 50019）

  • In-Sight 5600 シリーズのセンサから PC に大きなジョブファイルを保存できるようになりました。以前は、1.8MB 以上のジョブファイルを保存しようとした場合、In-Sight Explorer とセンサの通信が一時的に切断され、ジョブを PC に保存することができませんでした。（CR# 49835）

  • GetAllTime 関数は、結果キューを有効にした場合でも正しい値を返すようになりました。以前は、結果キューを有効にした場合、GetAllTime 関数は誤って小さな値を返していました。（CR# 49722）

  • In-Sight 接続マネージャは、ゲートウェイアドレスを 0.0.0.0 に設定した Microsoft® Windows® Server 2003 システム上で正常に機能するようになりました。以前は、In-Sight センサの MAC アドレスの入力後、In-Sight 接続マネージャが無応答になりました。（CR# 49826）

  • Windows インストーラ 3.0 のインストールされていない Windows Server 2003 上に In-Sight Explorer を正しくインストールすることができるようになりました。以前は、Windows インストーラ 3.0 のインストールされていない Windows Server 2003 上に In-Sight Explorer 3.4.0 をインストールすると、InstallShield 1603 エラーが発生していました。（CR# 49827）

  • CalibrateImage 関数は、インポートされた CalibrateGrid 関数のデータを参照しても適切な結果を表示できるようになりました。以前は、インポートされた CalibrateGrid データを参照している CalibrateImage 関数は #ERR を返していました。（CR# 46406）

  • ImageMath 関数が修正され、正しく表示されるようになりました（[ビジョンツール] → [画像フィルタ]）。以前は、関数が適切に表示されませんでした。（CR# 43885, 49630）

  • 結果キューの非表示状態がレイアウト保存されない問題を修正しました。以前は、[レイアウトを保存する] を実行しても、ジョブビューを開くたびに結果キューのフィルムストリップが表示されていました。（CR# 49266）

  • In-Sight 5000 シリーズのカラーセンサでも部分画像の取り組みができるようになりました。以前は、カラーセンサでは AcquireImage 関数の [取り込み開始行] および [行数] パラメータが無効となっていました。（CR# 36036）

  • EtherNet/IP の仕様に沿って、EtherNet/IP マルチキャストトラフィックの TTL が 7 から 1 に変更されました。この変更により、マルチキャストトラフィックは、単一のサブネットに制限されます。（CR# 50000）
