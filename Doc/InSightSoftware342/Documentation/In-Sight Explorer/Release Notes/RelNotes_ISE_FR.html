Notes de mise à jour du microprogramme In-Sight® Explorer 3.4.2

Présentation

La mise à jour 3.4.2 est une mise à jour de service du microprogramme/logiciel In-Sight Explorer. Cette mise à jour contient des modifications et des solutions à plusieurs problèmes connus, y compris des modifications aux boîtes de dialogue Paramètres Entrées discrètes et Paramètres Sorties discrètes afin de permettre la prise en charge du module d'extension E/S In-Sight 1400 et une mise à jour du microprogramme des capteurs In-Sight 5600.

La mise à jour de service 3.4.2 a été ajoutée à la mise à jour 3.4.1 du microprogramme/logiciel et contient les solutions fournies dans cette mise à jour. Cette mise à jour de service ne contient aucune nouvelle fonction. La version 3.4.0 présentait des nouvelles fonctions décrites dans les Notes de mise à jour du microprogramme In-Sight® Explorer 3.4.0. Pour des informations sur la configuration système requise, les problèmes connus et l'historique des mises à jour, consultez les Notes de mise à jour du microprogramme In-Sight® Explorer 3.4.0.

Pour obtenir les dernières notes de mise à jour, la documentation et autres, rendez-vous sur le Centre de support et d'apprentissage, à l'adresse : www.cognex.com/support/In-Sight.asp

Modifications et corrections de la mise à jour 3.4.2

REMARQUE : des numéros de demande de modification ( n° CR ) ont été inclus (lorsque cela est applicable) pour améliorer le suivi des problèmes connus, rapportés par l'intermédiaire du support technique de Cognex .

  • Les boîtes de dialogue Paramètres Entrées discrètes et Paramètres Sorties discrètes ont été modifiées afin de permettre la prise en charge du module d'extension E/S In-Sight 1400.

  • Le protocole de communications série des robots Kuka a été modifié de manière à envoyer une réponse DLE avant de traiter les commandes du mode natif. Auparavant, la réponse n'était envoyée qu'une fois la commande traitée, ce qui pouvait entraîner un dépassement du délai d'attente du robot lors d'une tentative de chargement d'un fichier de projet volumineux. (n° CR 49787)

  • Les capteurs In-Sight 5600 chargent désormais correctement les fichiers de projet volumineux qui contiennent certaines fonctions PatMax. Auparavant, il arrivait que le capteur se déconnecte du réseau et ne réponde plus lors d'une tentative de chargement d'un fichier de projet volumineux qui contenait plusieurs fonctions PatMax, obligeant à redémarrer le capteur. (n° CR 50762)

  • Les fonctions ScaleImage (Outils de vision --> Image) et VerifyText (Outils de vision --> OCV/OCR) ont été modifiées de façon à afficher l'image correcte lorsqu'elles sont utilisées en combinaison avec une acquisition partielle. Auparavant, lorsque ces fonctions étaient utilisées en combinaison avec une image partiellement acquise, l'image générée pouvait être corrompue. (n° CR 50931)

Modifications et corrections de la mise à jour 3.4.1

REMARQUE : des numéros de demande de modification ( n° CR ) ont été inclus (lorsque cela est applicable) pour améliorer le suivi des problèmes connus, rapportés par l'intermédiaire du support technique de Cognex .

  • Corrige un problème d'échec inattendu des déclencheurs d'acquisition des capteurs haute résolution In-Sight 5603 et 5403, lors de l'exécution de fonctions à fort pourcentage de calculs ou de mémoire en association avec l'utilisation des résultats d'inspection d'In-Sight Explorer. (n° CR 50393)

  • Sur les capteurs In-Sight série 5600, lors de l'exécution de fonctions à fort pourcentage de calculs, la corruption des images ne se produit plus. Autrefois, il arrivait que l'image devienne corrompue, et dans certains cas le capteur ne répondait pas. (n° CR 50385)

  • Corrige un problème qui concerne tous les capteurs In-Sight 5600 exécutant le microprogramme version 3.4.0. Si l'utilisateur avait totalement rempli et effacé le système de fichiers flash du capteur à plusieurs reprises, le risque que le capteur ne devienne inexploitable lors d'un prochain redémarrage est minime. Le microprogramme de tous les capteurs In-Sight 5600 devrait être mis à niveau vers la version 3.4.1. Ce problème n'affecte aucun autre modèle de capteurs In-Sight ni aucune autre version du microprogramme. (n° CR 49919)

  • La fonction Stringf a été améliorée pour retourner des chaînes de caractères informatives supplémentaires dans le tableur : la version du microprogramme ("%V"), le type de système auquel est connecté l'utilisateur ("% T "), l'adresse MAC ("% M ") et le numéro de série du capteur ("% N ").   ( n° CR 28899/44109)

  • Des profils supplémentaires RSLogix 5000 ont été implémentés pour faciliter l'intégration entre les capteurs de vision In-Sight Cognex et RSLogix 5000. Le programme d'installation se trouve sur le CD-ROM du logiciel In-Sight et mettra à jour les installations existantes de RSLogix 5000. La version 15.00 ou supérieure de RSLogix 5000 doit être installée pour pouvoir utiliser ces profils supplémentaires. ( n° CR 49914/42099)

  • Le paramètre Mode de la fonction ReadIDCode a été modifié et le paramètre Haute vitesse / Haut contraste est désormais correctement activé pour les codes UPC, EAN, DataMatrix, Code 128, Code 39, Code I2of5 et Code 93 tout en étant correctement désactivé pour les codes postaux. Lorsque la case Algorithme est cochée, la configuration du paramètre Mode sur Haute vitesse / Haut contraste peut accélérer les temps de décodage des codes UPC, EAN, DataMatrix, Code 128, Code 39, Code I2of5 et Code 93 s'ils sont bien imprimés. Auparavant, le paramètre Mode n'était pas activé pour ces types de code et incorrectement activé pour les codes postaux, ce qui n'améliorait en rien le temps de décodage lorsque le paramètre était configuré sur Haute vitesse / Haut contraste. ( n° CR 49842)

  • ValidateIDCode valide désormais correctement les codes DoDAAC contenant des caractères alphanumériques. Auparavant, ValidateIDCode signalait les codes DoDAAC contenant des caractères alphabétiques comme non valides. ( n° CR 49920)

  • La fonction LatchImage, lorsqu'elle est programmée dans l'émulateur, renvoie désormais le message #ERR si les réglages Hauteur et Largeur du paramètre Région font référence à une cellule vide. Auparavant, lorsque la fonction LatchImage était utilisée dans l'émulateur et qu'un des réglages Hauteur ou Largeur du paramètre Région faisait référence à une cellule vide, l'émulateur arrêtait de répondre. ( n° CR 50019)

  • In-Sight Explorer peut désormais enregistrer des fichiers de projet plus volumineux provenant d'un capteur de la série 5600 sur un PC. Auparavant, lors d'une tentative d'enregistrement d'un fichier de projet de plus de 1,8 Mo, In-Sight Explorer se déconnectait temporairement du capteur et le projet n'était pas enregistré sur le PC. (n° CR 49835)

  • La fonction GetAllTime retourne désormais la valeur correcte lorsque la Liste de résultats est activée. Auparavant, la fonction GetAllTime retournait une valeur incorrecte (inférieure) lorsque la Liste de résultats était activée. (n° CR 49722)

  • Le gestionnaire de connexion In-Sight fonctionne désormais correctement lorsqu'il est exécuté sur un système Microsoft® Windows® 2003 Server dont l'adresse de passerelle est définie sur 0.0.0.0. Auparavant, le gestionnaire de connexion In-Sight arrêtait de répondre lorsque l'adresse MAC d'un capteur In-Sight était saisie et appliquée. (n° CR 49826)

  • In-Sight Explorer s'installe désormais correctement sur un système Windows 2003 Server qui ne dispose pas du programme Windows Installer 3.0. Auparavant, une erreur InstallShield 1603 apparaissait lors de l'installation du logiciel In-Sight Explorer 3.4.0 sur un système Windows 2003 Server ne disposant pas de Windows Installer 3.0. (n° CR 49827)

  • La fonction CalibrateImage affiche désormais correctement les résultats lorsqu'elle fait référence à des données provenant d'une fonction CalibrateGrid importée. Auparavant, la fonction CalibrateImage renvoyait le message #ERR lorsqu'elle faisait référence à des données CalibrateGrid importées. (n° CR 46406)

  • La fonction ImageMath a été modifiée et s'affiche désormais correctement (Outils de vision --> Image). Auparavant, la fonction n'était pas correctement activée et n'était pas affichée. (n° CR 43885/49630)

  • La séquence vidéo Liste de résultats reste désactivée lorsqu'une Vue projet est rouverte quand l'affichage de la Liste des résultats est désactivé et que la mise en page de la Vue projet est enregistrée par le biais de l'option Enregistrer la mise en page. Auparavant, la séquence vidéo Liste de résultats était affichée à chaque fois qu'une Vue projet était ouverte, indépendamment de l'état de l'affichage après l'utilisation de l'option Enregistrer la mise en page. (n° CR 49266)

  • Les paramètres Première ligne et Nombre de lignes de la fonction AcquireImage renvoient désormais correctement une acquisition partielle s'ils sont utilisés sur des capteurs couleur de la série In-Sight 5000. Auparavant, ces paramètres étaient désactivés lors de l'utilisation d'un capteur couleur. (n° CR 36036)

  • Conformément à la spécification EtherNet/IP, le champ TTL du trafic de diffusion sélective EtherNet/IP a été changé de 7 à 1. Ce changement limite le trafic de diffusion sélective à un seul sous-réseau. (n° CR 50000)
